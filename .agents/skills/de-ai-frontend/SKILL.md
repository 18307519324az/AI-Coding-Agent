# De-AI Frontend

Goals:

- Make the interface feel like a practical developer SaaS console.
- Prefer restrained layout, real states, meaningful density, and specific product copy.
- Use navigation, tables, timelines, logs, diffs, test results, and approval controls where useful.

Avoid:

- Purple or blue gradient hero sections.
- Glassmorphism without product purpose.
- Empty marketing copy.
- Feature-card-only layouts.
- Identical rounded cards everywhere.
- Icon spam.

Require:

- Real navigation.
- Clear status indicators.
- Loading, empty, error, disabled, and success states.
- Accessible form labels.
- Specific copy tied to AI Coding Agent workflows.
- Playwright verification when UI behavior changes.

Before finishing:

1. Summarize UI changes.
2. List states covered.
3. Run lint and typecheck.
4. Run or explain Playwright checks.
